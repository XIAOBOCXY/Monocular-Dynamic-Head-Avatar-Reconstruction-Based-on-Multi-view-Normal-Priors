# 面向单目图像的多视角法线先验监督头部 Avatar 重建研究

## 摘要

单目图像驱动的三维头部 Avatar 重建属于神经渲染与生成式图形学交叉方向的重要问题，其目标是在仅给定一张人物头像图像的条件下，恢复可驱动、可重渲染、可进行表情迁移的三维头部表示。本文以 NeurIPS 2024 的 GAGAvatar 开源项目为基础，完成了本地环境复现、数据预处理、模型训练、推理验证与代码扩展，并围绕“几何监督不足导致重建边界与细节稳定性有限”这一问题，尝试引入 SOAP 生成的多视角法线先验作为额外几何约束。具体而言，本文首先在 VFHQ 子集上复现原始 GAGAvatar 的训练与推理流程；其次新增 matted 数据预处理工具，将裁剪后的人头图像进一步进行前景抠图并统一黑色背景；随后在模型配置、数据加载和损失函数层面扩展代码，使训练阶段能够读取 SOAP 离线生成的六视角法线图与掩码，并在预测双层 lifting plane 几何与 SOAP 法线之间加入余弦一致性损失。同时，为进行更完整的对照分析，本文还实现了 screen-space normal loss 作为中间对照组。

实验分别在两个本地数据子集上完成：其一为未做 matting 的 VFHQ group100_116 子集，其二为经过 matting 处理的 VFHQ group100_101 子集。定量结果表明，在 group100_116 上，screen normal loss 相比原始基线未带来明显提升，最佳 SSIM 由 0.5887 下降为 0.5869；在 group100_101_matted 上，不引入 SOAP 的基线模型达到最佳 PSNR 21.10、SSIM 0.6897，而从头训练的 SOAP 约束模型达到 PSNR 19.38、SSIM 0.6442，尚未优于基线。正在进行中的 SOAP 微调实验在 2000 step 时达到 SSIM 0.6639，早期收敛现象优于从头训练组，但仍需完整训练后才能形成最终结论。实验结果说明：matted 预处理对训练稳定性和背景一致性具有积极作用，而当前版本的 SOAP 法线伪监督尚未直接转化为更高的图像保真指标，原因可能与法线伪标签噪声、视角转换误差以及几何约束和图像重建目标之间的权衡有关。

本文的工作完整覆盖了高级计算机图形学课程作业对“复现、训练、改动代码、迁移到新流程并开展对比实验”的要求，属于在原始项目基础上的扩展性研究实践。

关键词：神经渲染；3D Gaussian Splatting；头部 Avatar；多视角法线先验；GAGAvatar；SOAP

## Abstract

Single-image 3D head avatar reconstruction is an important problem at the intersection of neural rendering and generative graphics. Its goal is to recover a controllable 3D head representation from a single portrait image for reenactment and novel-view rendering. This report is based on the open-source GAGAvatar project and extends the original pipeline with multi-view normal priors generated by SOAP. First, the original GAGAvatar training and inference pipeline was reproduced on a local Linux environment. Second, a matted data preprocessing pipeline was built to remove background inconsistency. Third, the codebase was modified at configuration, data loading, and loss computation levels so that the training procedure could consume offline SOAP six-view normal maps and masks. A cosine-consistency normal supervision term was then imposed between the predicted lifting-plane geometry and SOAP guidance. In addition, a screen-space normal loss was implemented as an intermediate ablation baseline.

Experiments were conducted on two local VFHQ subsets: an unmatted group100_116 subset and a matted group100_101 subset. On the unmatted subset, screen-space normal loss did not improve the baseline, with the best SSIM decreasing from 0.5887 to 0.5869. On the matted subset, the baseline model without SOAP reached PSNR 21.10 and SSIM 0.6897, while the SOAP-guided model trained from scratch reached PSNR 19.38 and SSIM 0.6442. An ongoing SOAP fine-tuning run reached SSIM 0.6639 at 2000 steps, which suggests better early convergence than training from scratch, but no final conclusion should be drawn before the full run finishes. The results indicate that matting is beneficial for background consistency and overall stability, whereas the current SOAP normal supervision has not yet translated into higher image-fidelity metrics. Possible reasons include noisy pseudo normals, view transformation error, and the trade-off between geometric consistency and image reconstruction quality.

Keywords: neural rendering; 3D Gaussian Splatting; head avatar; multi-view normal prior; GAGAvatar; SOAP

## 1 绪论

### 1.1 选题背景

近年来，神经渲染、三维重建和数字人生成成为高级计算机图形学的重要研究方向。相较于传统基于多视角图像或高精度扫描设备的数字人建模流程，单目图像驱动的三维头部重建具备更低的数据采集成本和更强的应用普适性，能够服务于虚拟主播、视频会议、短视频特效、人机交互、虚拟内容生产等场景。特别是在 3D Gaussian Splatting 兴起之后，基于高斯表示的头像重建与实时驱动方法兼具较好的渲染质量和推理速度，成为近期研究热点。

GAGAvatar 是这一方向中较有代表性的工作之一。其核心目标是从单张输入头像图像中，重建可进行表情与姿态迁移的头部 Avatar，并通过高斯渲染实现实时驱动。该方法避免了多帧视频或多视角采集需求，具有较高的实用价值。然而，单图像重建在几何信息上天然存在歧义，特别是头发边界、帽檐、侧脸轮廓等区域，往往只能通过隐式先验或弱监督约束恢复，容易出现几何漂移与边界模糊。

因此，本文围绕“如何为单目头部重建提供更强的几何先验”展开实践研究，尝试引入 SOAP 生成的多视角法线图作为额外监督，希望通过离线生成的伪多视角几何信息，增强训练过程中的三维一致性约束。

### 1.2 研究意义

本课题的研究意义主要体现在以下三个方面。

第一，从课程要求角度看，本项目并非停留在开源代码下载与运行示例阶段，而是在完成环境复现和训练流程跑通的基础上，对损失函数、数据流和工程脚本进行了实质性修改，并在新数据流程上重新训练与微调，符合高级计算机图形学课程对高层级任务的要求。

第二，从技术角度看，GAGAvatar 原始训练目标主要围绕感知损失、图像重建损失、包围框约束与点集约束展开。本文引入的多视角法线监督，属于将外部几何生成模型的输出融入头部 Avatar 重建的尝试。这一思路虽然当前尚未获得更优的 PSNR 与 SSIM，但它提供了一条将生成式多视角先验与神经渲染训练结合的实验路径。

第三，从研究训练角度看，负结果本身同样具有价值。研究生阶段的课程报告不应只呈现“成功提升”的结论，更应说明在什么条件下某种方法没有达到预期，以及可能的原因与后续改进空间。本文的实验恰好反映出：外部法线伪监督并不天然等价于更高图像质量，如何进行空间对齐、权重调节和训练策略设计，仍然值得进一步研究。

### 1.3 本文工作与贡献

本文的主要工作与贡献可以概括为以下四点。

1. 在本地 Linux 环境中完成 GAGAvatar 项目的复现，跑通预训练推理、数据集训练与模型验证流程。

2. 构建了面向本地 VFHQ 子集的数据预处理流程，包括 LMDB 数据抠图、黑背景统一以及 SOAP 特征帧导出。

3. 对原始代码进行扩展，实现了两类法线监督：一类是 screen-space normal loss，用于同一视角下的法线一致性对照实验；另一类是 SOAP 多视角法线监督，用于将 SOAP 离线生成的六视角法线图接入训练。

4. 基于 5 组实验记录完成定量与定性对比分析，指出当前版本的主要有效因素是 matting 预处理，而 SOAP 法线先验在当前实现下尚未超过 matted 基线，同时给出后续改进方向。


## 2 相关方法与总体流程

### 2.1 GAGAvatar 基线方法

GAGAvatar 是一个基于 3D Gaussian 表示的单图像头部 Avatar 重建框架。其核心思想可以概括为：首先通过跟踪模块从输入头像中恢复 FLAME 参数与相机变换；随后从输入图像中提取语义与外观特征；再基于 canonical template 与 lifting plane 生成高斯参数；最终通过高斯渲染和超分网络生成目标驱动图像。

从实现上看，原始模型主要由以下几个模块构成。

1. 输入特征编码模块。代码中使用 DINOv2 提取图像表征，用于驱动后续几何与外观生成。

2. 几何生成模块。模型同时利用 FLAME 点云与双层 lifting plane 生成高斯表示，兼顾基础面部结构与发型、帽子等复杂外观。

3. 渲染模块。先通过 Gaussian renderer 得到中间渲染结果，再由 StyleUNet 进行超分重建，输出最终图像。

4. 损失函数模块。原始训练目标由感知损失、像素重建损失、包围框约束与点集距离约束构成。

原始损失可以写为：

$$
\mathcal{L}_{base}=\mathcal{L}_{percep}+\mathcal{L}_{img}+\mathcal{L}_{box}+\mathcal{L}_{point}
$$

其中，$\mathcal{L}_{percep}$ 强调感知一致性，$\mathcal{L}_{img}$ 对应像素级图像重建，$\mathcal{L}_{box}$ 用于约束头部区域范围，$\mathcal{L}_{point}$ 用于约束预测点与目标点的空间距离。

### 2.2 SOAP 多视角法线先验

SOAP 全称为 Style-Omniscient Animatable Portraits，是一个面向可动画头像的多视角生成与重建方法。SOAP 可以从单张肖像图像出发，生成多视角人头图像、法线图以及相关几何信息。与 GAGAvatar 直接从单图学习重建不同，SOAP 提供的是一种离线多视角先验，能够为训练阶段提供额外的几何约束。

本文并没有将 SOAP 作为最终重建器替换 GAGAvatar，而是把 SOAP 视为一个“教师先验生成器”。具体做法是：先从训练集中为每个视频序列导出 canonical feature frame，再运行 SOAP 离线生成 6 个固定视角下的图像与法线图，最后在 GAGAvatar 训练阶段读取这些结果，对预测的 lifting plane 几何做法线一致性监督。

### 2.3 整体流程设计

本文整体流程如下。

1. 以 VFHQ 子集为输入，构建 img_lmdb、optim.pkl 与 dataset.json。

2. 对原始 512×512 头像裁剪图做 matting，生成黑背景版本，缓解训练与推理时的背景域差异。

3. 从每个视频序列导出 canonical feature frame，作为 SOAP 的输入。

4. 运行 SOAP，得到固定六视角对应的 RGB 图像与法线图。

5. 在训练阶段由数据加载器读取 SOAP 图像、法线图与前景 mask。

6. 在模型内部把预测的双层 lifting plane 顶点渲染为法线图，并与 SOAP 法线图在前景重叠区域上计算余弦一致性损失。

7. 通过验证集上的 PSNR 与 SSIM 对不同实验设置进行比较。

加入 SOAP 后，总损失可写为：

$$
\mathcal{L}=\mathcal{L}_{base}+\lambda_n\mathcal{L}_{normal}+\lambda_s\mathcal{L}_{soap-normal}
$$

其中，screen normal loss 实验对应 $\lambda_n>0,\lambda_s=0$，SOAP 主实验对应 $\lambda_n=0,\lambda_s>0$。当前主实验中 $\lambda_s=0.05$。

SOAP 法线项的核心形式可以写为：

$$
\mathcal{L}_{soap-normal}=\frac{1}{|\Omega|}\sum_{p\in\Omega}\left(1-\left|\langle \hat{n}_p,n^{soap}_p\rangle\right|\right)
$$

其中，$\Omega$ 表示预测法线与 SOAP 法线前景重叠区域，$\hat{n}_p$ 为模型预测法线，$n^{soap}_p$ 为 SOAP 提供的目标法线。


## 3 实验环境、数据集与评价指标

### 3.1 硬件与软件环境

7. 其他关键依赖：Lightning、Torchvision、FAISS-GPU、TensorBoard

### 3.2 数据集与预处理

本文使用的是本地 VFHQ 数据子集，而非完整公开训练集。实验中实际涉及两个数据目录。

1. vfhq_group100_116：未进行 matting 处理的数据子集，用于原始基线与 screen normal loss 对照实验。其数据划分为 train 82346、val 10094、test 0。

2. vfhq_group100_101_matted：经过 matting 处理的数据子集，用于 matted 基线、SOAP 从头训练与 SOAP 微调实验。其数据划分为 train 8898、val 64、test 32。

预处理流程如下。

1. 原始头像按照 GAGAvatar 官方流程裁剪为 512×512 的头部区域。

2. 使用新增脚本对 LMDB 中图像逐帧进行人像抠图，并统一为黑色背景，减少背景噪声与训练推理域差异。

3. 由 tracked 数据中每个视频序列选取 canonical feature frame，并导出为 PNG 图像，供 SOAP 处理。

4. 运行 SOAP 后，在每个样本目录下得到六视角 images 与 normals，用作离线教师监督。

需要说明的是，group100_116 与 group100_101_matted 并非相同数据子集，因此跨子集结果不能直接用来得出严格因果结论。本文的公平对比主要在同一数据子集内部进行。

### 3.3 训练配置与评价指标

训练主要参数如下。

1. 优化器：Adam

2. 初始学习率：1e-4

3. Batch size：4

4. 训练步数：20000

5. 验证间隔：每 2000 step 验证一次

6. 焦距参数：12.0

7. point plane size：296

评价指标采用 PSNR 与 SSIM，两者均由验证集统一计算。其中，PSNR 反映像素重建误差，SSIM 更强调结构相似性。在本项目中，SSIM 也是 best checkpoint 的保存依据。


## 4 代码修改与实现说明

### 4.1 配置层改动

配置层面的核心修改集中在两个文件。

第一，在 configs/model/gaga.yaml 中新增 NORMAL_LOSS 与 SOAP_GUIDANCE 两个配置块，并将训练设置从官方默认的较大规模调整为更适合本地实验的 batch size 4、训练 20000 step、每 2000 step 验证一次。这样做的作用有两点：一是便于进行多组对照实验；二是使损失开关完全配置化，避免频繁手改模型源码。

第二，在 configs/data/vfhq.yaml 中把数据路径改到本地子集路径，并新增 DATASET.SOAP_GUIDANCE 配置项，包括 SOAP 根目录、加载分辨率、是否固定使用 canonical feature frame 等。这样一来，训练阶段可以通过配置决定是否启用 SOAP 指导，而不需要再额外改数据集代码。

### 4.2 数据加载层改动

数据加载层的主要修改位于 core/data/loader_track.py，具体可概括为以下四点。

1. 新增 SOAP 配置解析逻辑。数据集初始化时读取 ROOT、LOAD_SIZE、USE_CANONICAL_FEATURE_FRAME 等参数，并在启用 SOAP 时检查路径合法性。

2. 修改特征帧选择策略。原始训练流程中，feature frame 可能从同一视频随机抽样；加入 SOAP 后，为了让 SOAP 路径与训练样本稳定对应，代码支持固定使用每个视频序列的 canonical feature frame。

3. 加载 SOAP 监督数据。新增 _load_soap_guidance 函数，从指定目录读取多视角 images、normals 与 masks，并在 _load_one_record 中一起返回给模型。

4. 修复视频 ID 解析逻辑。原代码对带前缀的数据命名存在分组风险，本文将 get_video_id 改为基于 frame_key.rsplit('_', 1)[0] 的方式解析视频键，避免错误把不同帧归并到同一视频或错误匹配 SOAP 路径。

这部分改动非常关键，因为 SOAP 指导并不是简单增加一个损失函数，而是需要先把多视角监督数据真正接入训练数据流。

### 4.3 模型与损失函数改动

模型层改动主要位于 core/models/GAGAvatar/models.py，是本次扩展的核心。主要工作包括。

1. 新增 NORMAL_LOSS 与 SOAP_GUIDANCE 的配置读取与初始化逻辑，支持在模型构造时决定是否启用法线监督。

2. 引入 PyTorch3D 的 MeshRasterizer、PerspectiveCameras、interpolate_face_attributes 等组件，用于将预测顶点与 FLAME 顶点渲染为法线图。

3. 新增 point normal loss。该方式先为双层 lifting plane 网格估计顶点法线，再与最近邻匹配到的 FLAME 顶点法线做余弦一致性约束。该分支为更轻量的三维法线监督方案。

4. 新增 screen-space normal loss。该方式把预测的双层 plane 顶点与目标 FLAME 顶点都渲染到同一目标视角下，生成 normal map，并在屏幕空间上计算重叠区域的一致性损失。Apr24_1809_mqxdz 实验使用的是这一分支。

5. 新增 SOAP normal guidance。该方式基于设定的六个方位角构造虚拟相机，把预测的 lifting plane 几何渲染为多视角法线图，再和 SOAP 生成的多视角法线图在前景区域上计算余弦一致性损失。Apr25_1030_kxwey 与 Apr25_1707_nheku 使用的就是这一分支。

6. 保留 SOAP RGB guidance 接口。虽然当前实验中 SOAP_RGB_WEIGHT 设为 0.0，没有实际参与优化，但代码中已预留 RGB 引导分支，便于后续进一步扩展。

换言之，你这次报告里最准确的表述不是“只加了一个 screen loss”，而是“实现了两类几何监督，其中主方法是 SOAP 多视角法线先验监督，screen loss 属于对照实验”。

4. 从推理结果来看，模型已经具备较好的单图驱动能力，即输入一张特征人脸图与一张驱动图或驱动视频后，可以输出符合目标姿态与表情变化的头部结果，说明整体系统功能是完备的。

### 5.4 结果讨论

根据当前实验，可以得到以下讨论结论。

首先，matting 很可能是本次实验里最有效的工程改动。虽然 E1 与 E3 跨数据子集，不能直接作为严格因果证据，但从整体结果幅度来看，matted 数据对应的验证指标明显更高，而且定性图中背景边界更干净。这与人像重建任务对前景背景分布敏感的经验一致。

其次，screen-space normal loss 没有在无 matting 数据上带来收益。可能原因在于：当背景较杂乱、轮廓本身存在不稳定时，仅靠屏幕空间法线约束不足以提供足够干净的几何监督，反而可能引入额外优化冲突。

再次，SOAP 多视角法线先验从头训练未超过基线，说明外部法线伪标签并不天然等价于更好的重建质量。本文认为可能存在以下原因。

1. SOAP 生成法线图本质上仍然是伪监督，其几何精度与 GAGAvatar 内部坐标系之间并非完全一致。

2. 训练中只对 lifting plane 几何做多视角法线监督，而最终图像质量仍然受高斯参数分布、外观纹理、超分网络等多因素影响。

3. 当前 SOAP 法线项权重固定为 0.05，未进行更细致的权重搜索或分阶段调度，可能在早期训练中过强或过弱。

4. 当前从头训练的方式可能使模型同时适应图像重建目标和新几何约束，优化更困难。E5 的早期结果反而提示，基于已收敛基线再加入 SOAP 约束的微调策略可能更合理。

因此，本文的结论并不是“SOAP 方法无效”，而是“当前实现与当前训练设置下，SOAP 多视角法线约束尚未转化为更优的 PSNR/SSIM，需要进一步改进对齐方式、监督策略和训练流程”。这一结论本身也具有研究价值。

本文围绕单目图像驱动的三维头部 Avatar 重建任务，以 GAGAvatar 为基线完成了较完整的复现与扩展实践。具体来说，本文先完成了项目环境配置、预训练模型推理和本地训练流程验证，随后对代码进行了多处改动，包括数据集 matting 预处理、SOAP 输入帧导出、数据加载扩展、screen-space normal loss 实现、SOAP 多视角法线监督实现、训练日志 TensorBoard 记录等。实验部分在 5 组设置上进行了对比，较系统地分析了不同监督方式的作用。

从实验结果看，当前最优完成实验来自 matted 数据上的原始基线模型，其最佳指标为 PSNR 21.10、SSIM 0.6897。screen-space normal loss 未在未抠图数据上带来明显收益；SOAP 多视角法线约束的从头训练版本尚未优于 matted 基线，但正在进行中的微调实验在早期阶段表现出更好的收敛趋势。这说明本文提出的研究方向具有探索价值，但当前仍处于初步实现阶段。

后续工作可从以下几个方向继续推进。

1. 在同一数据子集上做更严格的全量对照，包括“无 matting”“有 matting”“有 matting + screen normal”“有 matting + SOAP”四组完整实验，避免子集变化干扰结论。

2. 进一步检查 SOAP 法线的相机坐标系、法线方向与掩码区域定义，提升监督信号的一致性。

3. 尝试分阶段训练策略，例如先训练基线模型，再逐步增加 SOAP 权重，而不是从头直接加入法线约束。

4. 探索同时启用 SOAP RGB guidance 与 normal guidance，或引入更稳定的多视角一致性损失。

5. 若时间允许，可把实验迁移到更接近课程“新任务”的设定，例如不同风格肖像数据、动漫头像或自拍视频数据，以进一步突出改造与迁移能力。

单目图像驱动的三维头部 Avatar 重建是神经渲染与生成式图形学中的重要问题，其目标是在仅给定单张人脸图像的条件下重建可驱动、可渲染、可重定向的三维头部表示。本文基于 NeurIPS 2024 的 GAGAvatar 开源项目完成环境复现、训练复现与本地数据实验，并进一步面向几何一致性不足的问题，引入 SOAP 提供的多视角法线先验监督，对原始训练框架进行了扩展。具体而言，本文首先完成了 VFHQ 子集的本地训练与推理流程，随后新增了 LMDB 数据集 matting 预处理脚本、SOAP 特征帧导出脚本，并在数据加载器中加入 SOAP 六视角图像、法线与掩码的读取逻辑；在模型侧，新增了屏幕空间法线损失与 SOAP 多视角法线监督损失，实现了从二维图像重建到几何先验约束的训练扩展。实验分别在未做 matting 的 VFHQ group100_116 子集和做过 matting 的 VFHQ group100_101 子集上进行。结果表明：在 group100_116 上，单独加入 screen normal loss 并未优于基线；在 group100_101_matted 上，从头训练加入 SOAP 约束的结果低于 matted 基线，但微调实验在早期阶段表现出更快的收敛趋势。综合来看，matting 预处理对结果稳定性与背景质量帮助更明显，而当前 SOAP 法线监督仍受伪法线噪声、视角不一致与损失权重设置影响，尚未转化为最终 PSNR/SSIM 的提升。本文工作完成了“开源代码复现 + 本地训练 + 方法修改 + 对比实验分析”的完整闭环，满足高级计算图形学课程高层级任务要求。

**关键词**：神经渲染；三维头部 Avatar；3D Gaussian Splatting；法线先验；GAGAvatar；SOAP

## Abstract

Single-image 3D head avatar reconstruction is an important problem in neural rendering and generative graphics. Its goal is to reconstruct a controllable and animatable 3D head representation from only one portrait image. In this work, I first reproduced the open-source GAGAvatar framework, including environment setup, inference, and local training on VFHQ subsets. Then I extended the original training pipeline by introducing multi-view normal prior supervision generated by SOAP, aiming to improve geometry consistency during reconstruction. Specifically, I added an LMDB matting preprocessing script, a canonical feature-frame export tool for SOAP, SOAP guidance loading logic in the dataset loader, and two geometry-related supervision branches in the model: screen-space normal loss and SOAP multi-view normal guidance loss. Experiments were conducted on two local VFHQ subsets: the un-matted group100_116 subset and the matted group100_101 subset. Results show that screen normal loss does not outperform the baseline on group100_116. On the matted group100_101 subset, training from scratch with SOAP guidance underperforms the matted baseline, while the fine-tuning experiment shows faster early-stage convergence but has not finished yet. Overall, matting preprocessing brings clearer gains in stability and visual cleanliness, whereas the current SOAP-based normal supervision is still limited by pseudo-normal noise, view inconsistency, and loss-weight sensitivity. This work completes the full pipeline of reproduction, local training, code modification, and comparative analysis, which meets the advanced-level requirement of the Advanced Computer Graphics course.

## 2 相关技术与总体方案

### 2.1 GAGAvatar 基线方法

GAGAvatar 是一个面向单图像三维头部 Avatar 重建与驱动的框架。其核心思想可以概括为：先借助跟踪模块从输入图像中提取 FLAME 参数、相机位姿与关键点信息，再利用视觉编码器提取外观特征，随后通过双层 lifting planes 与高斯表示生成可以渲染的三维头部模型，最后在目标表情和姿态驱动下生成目标视角图像。

从实现角度看，原始训练主要依赖以下损失项。

1. 感知损失，用于约束输出图像与目标图像在人脸感知空间上的一致性。
2. 图像重建损失，用于约束像素级结果。
3. 边界框损失，用于保持人脸区域与裁剪框对应关系。
4. 点集损失，用于约束预测几何点与 FLAME 目标点之间的距离。

其优点是能够在较弱监督条件下学习出可驱动头部表示，但不足也较明显：当图像区域存在弱纹理、大面积头发、帽子、遮挡或光照变化时，仅靠二维图像监督未必足以保证几何正确。

### 2.2 SOAP 多视角先验

SOAP 是一个面向风格一致、可驱动人像重建的多视角生成与重建框架。SOAP 的处理流程可以从单张输入图像生成多视角图像与法线图，这为本文提供了一种可离线获取“伪几何监督”的思路。本文并未将 SOAP 作为主重建器使用，而是将其作为 teacher prior：先对特征帧运行 SOAP，得到 6 个视角下的 normal maps，再将这些 normal maps 作为 GAGAvatar 的辅助监督信号。

这类监督的优点在于，不需要额外的真实多视角采集设备，即可利用已有生成模型提供几何线索；缺点则在于 SOAP 生成的法线本身并非真实标注，存在视角偏差、外观幻觉和边界噪声，因此它更适合作为“软约束”，而不是绝对真值。

### 2.3 本文总体流程

本文整体流程如下。

1. 从 VFHQ 子集中构建 LMDB 数据集，并获得官方跟踪所需的 optim.pkl 与 dataset.json。
2. 对部分数据集执行 matting，使训练背景接近推理时的黑背景裁剪头部图像。
3. 从每个视频的 canonical feature frame 导出特征帧。
4. 将导出的特征帧输入 SOAP，离线得到 6 视角图像、normal maps 与前景信息。
5. 修改 GAGAvatar 的数据加载器，使其在训练阶段能够加载 SOAP 结果。
6. 修改 GAGAvatar 的模型与损失函数，在原始损失基础上引入 normal guidance。
7. 进行基线、screen loss、SOAP scratch、SOAP fine-tune 等多组实验比较。

本文的总损失可写为：

$$
\mathcal{L}=\mathcal{L}_{percep}+\mathcal{L}_{img}+\mathcal{L}_{box}+\mathcal{L}_{point}+\lambda_n\mathcal{L}_{normal}+\lambda_s\mathcal{L}_{soap}
$$

其中，screen loss 实验中 $\lambda_n=0.05, \lambda_s=0$；SOAP 主实验中 $\lambda_n=0, \lambda_s=0.05$。本文当前实现中 SOAP 约束主要采用法线一致性项，其核心形式为：

$$
\mathcal{L}_{soap}=\frac{1}{|\Omega|}\sum_{p\in\Omega}\left(1-|\langle \hat{n}_p,n^{soap}_p\rangle|\right)
$$

其中 $\Omega$ 表示预测前景与 SOAP 前景的重叠区域，$\hat{n}_p$ 为模型渲染得到的法线，$n^{soap}_p$ 为 SOAP 提供的目标法线。

## 4 代码修改与实现细节

### 4.1 配置层修改

本文首先修改了模型配置与数据配置文件，使原始代码可以切换不同实验模式。

#### 4.1.1 模型配置修改

在 `configs/model/gaga.yaml` 中新增了两组配置。

1. `NORMAL_LOSS`：用于开启普通法线损失，支持 `point` 与 `screen` 两种模式。
2. `SOAP_GUIDANCE`：用于开启 SOAP 多视角监督，包含法线权重、RGB 权重、渲染尺寸、俯仰角与多视角列表。

同时，原始配置中的训练规模为 batch size 8、200000 iter。考虑到本地课程实验的时间成本，本文将训练改为 batch size 4、20000 iter、每 2000 步验证一次。这样做虽然降低了训练总量，但能在有限算力与时间内快速观察不同监督项的变化趋势。

#### 4.1.2 数据配置修改

在 `configs/data/vfhq.yaml` 中，本文将原始官方数据路径切换为本地数据路径，并新增 `DATASET.SOAP_GUIDANCE` 字段，用于指定 SOAP 结果根目录、读取尺寸以及是否固定使用 canonical feature frame。该修改使训练脚本能够直接区分“普通训练”与“SOAP 引导训练”。

### 4.2 数据加载层修改

数据加载层的核心修改集中在 `core/data/loader_track.py`，这是本次项目中最关键的改动之一。

第一，新增了 SOAP 配置读取逻辑。当 `SOAP_GUIDANCE.ENABLED=True` 时，数据加载器会自动读取 SOAP 根目录，并检查配置是否完整。

第二，修改了 feature frame 的选择策略。原始训练中，feature image 会从同视频中随机采样；而为了与 SOAP 结果对应，本文在启用 SOAP 时优先固定使用 canonical feature frame，这样每个视频的 SOAP 指导都能稳定对应到同一个关键帧。

第三，新增了 `_load_soap_guidance()`，负责读取离线生成的多视角图像、法线图与 mask，并在训练时组装为 batch 字段供模型使用。

第四，修复了视频 ID 的解析逻辑。原始实现中，`frame_key.split('_')[0]` 在带前缀命名的数据下会把不同视频错误合并。本文将其改为 `frame_key.rsplit('_', 1)[0]` 并兼容 `img_` 前缀，从而保证 feature frame 与 SOAP 路径索引的一致性。这个修复不仅关系到 SOAP 路径能否正确匹配，也关系到“同视频采样”本身是否正确。

### 4.3 模型与损失层修改

模型侧的主要修改位于 `core/models/GAGAvatar/models.py`。

#### 4.3.1 法线损失总控开关

在 `__init__` 中，本文新增了对 `NORMAL_LOSS` 与 `SOAP_GUIDANCE` 的解析逻辑，并根据配置决定是否初始化额外渲染器、FLAME 面片和视角设置。

#### 4.3.2 point normal loss 与 screen normal loss

为便于对比，本文实现了两类法线约束。

1. point normal loss：对双层 lifting plane 的点云估计法线，并与目标 FLAME 网格顶点法线进行匹配。
2. screen normal loss：将预测几何与目标 FLAME 同时渲染到同一屏幕空间 normal map 上，在重叠区域做余弦一致性约束。

其中，screen loss 更贴近图像域监督，也是本文在未做 matting 数据上首先测试的版本。

#### 4.3.3 SOAP multi-view normal guidance

当开启 `SOAP_GUIDANCE` 时，模型会执行以下步骤。

1. 根据 feature frame 的相机外参构造 6 个 SOAP 视角。
2. 将预测的双层 plane 几何渲染成多视角 normal maps。
3. 对 SOAP normal maps 做尺寸对齐与方向旋转修正。
4. 在预测前景与 SOAP 前景的重叠区域上计算法线余弦一致性损失。

实现上，本文为此新增了 `_calc_soap_guidance_loss()`、`_calc_soap_normal_guidance()`、`_build_soap_view_transforms()`、`_rotate_soap_normals()` 等函数。为支持这些操作，还扩展了辅助函数，例如 `build_plane_faces()`、`build_yaw_rotations()`、`build_cameras()` 等。

从代码逻辑上看，这意味着本文已经不再是“单纯改参数”，而是实质上为 GAGAvatar 增加了一条新的几何监督分支。

### 4.4 数据工具链与训练监控修改

除了模型与数据加载层，本文还补充了多个支撑实验的工具文件。

| 文件 | 修改类型 | 作用 |
| --- | --- | --- |
| `tools/matte_lmdb_dataset.py` | 新增 | 对已有 LMDB 数据做 matting，支持复制或重跑 tracking |
| `tools/export_soap_feature_frames.py` | 新增 | 从数据集中导出 canonical feature frame 给 SOAP 使用 |
| `tools/merge_tracked_datasets.py` | 新增 | 合并多个 tracked 数据集，便于本地构造实验集 |
| `train.py` | 修改 | 增加 TensorBoard 日志、训练/验证标量记录与图像记录 |
| `core/models/modules/dino_base.py` | 修改 | 本地优先加载 DINOv2，避免网络依赖 |
| `build_resources.sh` | 修改 | 使用 hf-mirror 下载资源，提升国内环境可用性 |

其中，`matte_lmdb_dataset.py` 与 `export_soap_feature_frames.py` 是本次课程作业中最有价值的工程化补充，因为它们直接支撑了“数据预处理 -> 外部先验生成 -> 训练接入”的新方法链路。

### 5.3 定性结果分析

从验证图与推理图可以观察到如下现象。

第一，matted 数据训练得到的结果背景更干净，人物边界更稳定。在验证图中，黑背景与头部边界的过渡更自然，这与训练-推理背景分布更加一致有关。

第二，SOAP 从头训练虽然在整体结构上仍能完成重演，但在头发、帽檐、面部边缘等位置更容易出现轻微模糊、边界泛化或几何过平滑的现象。这说明 SOAP 法线先验虽然提供了几何信息，但当前引入方式会抑制一部分原本有利于二维重建的自由度。

第三，推理图表明模型已经具备“输入单张静态人像 + 目标驱动图像/视频 -> 输出重演头像”的完整能力，说明项目在功能上已经超过仅运行 demo 的基础层级要求。

### 5.4 结果讨论

#### 5.4.1 为什么 screen loss 没有带来提升

可能原因包括以下几点。

1. screen normal loss 的监督目标来自 FLAME 渲染，而 lifting planes 本身并不一定严格对齐 FLAME 表面，导致监督存在结构偏差。
2. 在未做 matting 的复杂背景数据上，图像重建误差和背景干扰更强，几何法线损失难以主导优化方向。
3. 当前法线权重为 0.05，可能仍不足以显著改变主损失趋势，或者在某些阶段过强、在某些阶段过弱。

#### 5.4.2 为什么 SOAP 从头训练反而更差

这是本文最重要的实验结论之一，也是报告中应当诚实强调的部分。

1. SOAP normal map 并非真实多视角采集法线，而是由外部生成流程推断得到，存在伪标签噪声。
2. SOAP 视角与 GAGAvatar 内部相机空间之间虽然做了旋转补偿，但仍可能存在尺度、法线方向或头部姿态不完全对齐的问题。
3. 从头训练阶段，模型本身的几何尚未稳定，如果过早加入额外几何约束，容易造成优化目标冲突。

#### 5.4.3 为什么微调结果值得继续跑完

尽管 E5 尚未完成，但它在 2000 步时已经达到 0.6639 的 SSIM，说明“先学会重建，再加入 SOAP 约束”可能是更合理的训练策略。换言之，SOAP 可能更适合作为后期细化约束，而不适合作为一开始就参与优化的主监督。这一观察为后续改进提供了方向。

## 6 总结与展望

本文围绕单目图像三维头部 Avatar 重建任务，基于 GAGAvatar 完成了从环境搭建、训练复现、推理验证到方法修改和对比实验分析的完整流程。与只运行官方示例不同，本文针对原始方法几何约束不足的问题，新增了 matting 数据预处理流程、screen normal loss、SOAP 多视角法线监督以及一整套离线 SOAP 数据接入链路，在代码、数据和实验三个层面都进行了实质性扩展。

从当前实验结果看，最明确的正向结论是：matting 预处理对训练稳定性、背景清洁度和整体图像质量具有明显帮助。相较之下，screen normal loss 与 SOAP 从头训练尚未带来最终指标提升，但它们提供了有价值的反例和经验：外部几何先验并不是“接入后自然增益”，其效果高度依赖先验质量、空间对齐方式、权重设置和训练阶段。

后续工作可以从以下几个方向继续推进。

1. 在相同数据子集上补齐“无 matting / 有 matting / +screen / +SOAP”的完全公平对比，避免子集差异影响结论。
2. 将 SOAP 约束改为分阶段启用，例如先训练到稳定收敛，再逐步增加法线权重。
3. 进一步校正 SOAP normal 与 GAGAvatar 相机空间之间的几何对齐关系，降低伪监督误差。
4. 尝试引入 SOAP RGB guidance、轮廓约束或边缘权重，而不仅仅使用法线余弦损失。
5. 在更多身份与更长驱动视频上做泛化测试，验证方法是否具有稳定可迁移性。

总体而言，本文已达到课程“高层级”任务要求：不仅完成开源代码复现与本地训练，还对原始方法进行了明确的代码修改，并在新设置下完成了重新训练、微调和对比分析。
