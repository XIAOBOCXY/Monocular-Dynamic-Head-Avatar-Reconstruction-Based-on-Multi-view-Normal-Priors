## 一、作品简介

### 1. 任务背景与应用场景

本作品属于计算机视觉、神经渲染与三维生成的交叉方向，目标是从单张人脸图像中重建一个可驱动的三维头部 Avatar，并在目标驱动图像或视频的控制下生成新的表情与姿态结果。该任务可应用于数字人驱动、虚拟主播、人像视频生成、远程会议、短视频内容生成与人机交互等场景。

与传统二维人脸编辑相比，三维头部 Avatar 重建能够更好地保持视角变化、表情驱动与头部结构的一致性，因此具有更强的实用价值。本文基于 GAGAvatar 项目完成本地训练与推理，并进一步加入 SOAP 多视角法线先验监督，用于探索外部几何先验对模型训练的影响。

### 2. 核心功能

本作品实现了以下核心功能。

1. 数据集加载与预处理：支持 VFHQ 子集的 LMDB 训练数据读取，并新增 matting 预处理与多数据集合并工具。
2. 深度学习模型构建：基于 GAGAvatar 实现单图像三维头部重建与重演；在模型侧新增 screen normal loss 与 SOAP normal guidance。
3. 模型训练、验证与测试：支持从头训练与微调，训练过程中自动保存 best checkpoint。
4. 结果评估、指标计算与可视化：记录 PSNR、SSIM，并保存验证拼图和 TensorBoard 日志。
5. 模型对比实验：完成 5 组实验，包括基线、screen loss、matted 基线、SOAP 从头训练和 SOAP 微调。

### 3. 所使用的技术

1. 编程语言：Python
2. 框架：PyTorch、PyTorch3D、Lightning
3. 主要工具库：Torchvision、FAISS-GPU、NumPy、TensorBoard、TQDM
4. 硬件环境：NVIDIA GeForce RTX 4090 24GB，CUDA 12.1

## 二、相关技术原理

### 1. 模型结构与核心思想

本作品的基础模型为 GAGAvatar。它并不是传统意义上的单纯 CNN 分类网络，而是由图像编码、参数化人脸几何、双层几何平面表示和高斯渲染组成的复合系统。

1. 图像编码部分：使用 DINOv2 提取输入人像的外观特征。
2. 参数化几何部分：使用 FLAME 作为人脸几何先验，为训练提供姿态、表情和几何目标点。
3. 可学习表示部分：使用双层 lifting planes 与 3D Gaussian 参数表示三维头部结构。
4. 渲染部分：使用高斯渲染器和上采样网络输出高分辨率重演图像。

其核心思想是：利用单张图像中的外观信息与三维人脸先验共同驱动一个可渲染的三维表示，从而在目标驱动条件下生成新的人脸图像。

### 2. 关键机制说明

#### 2.1 FLAME 几何先验

FLAME 是参数化三维人脸模型，能够通过 shape、pose、expression、eye pose 等参数控制头部几何。本文使用 FLAME 提供目标点集与法线参考，是训练几何对齐的重要基础。

#### 2.2 3D Gaussian Splatting 风格渲染

GAGAvatar 使用高斯表示对头像进行建模，这类表示相比传统网格贴图更适合神经渲染任务，具有连续、可微和推理较快的优点。

#### 2.3 SOAP 多视角法线先验

SOAP 能从单张图像生成 6 个视角的图像和法线图。本文将其作为外部 teacher prior，把这些法线图作为额外几何监督，从而尝试改善模型在头发、侧脸和边界区域的几何表达能力。

### 3. 损失函数与优化器

本作品使用 Adam 优化器，学习率为 1e-4。总损失由以下几部分组成。

1. 感知损失 `percep_loss`
2. 像素重建损失 `img_loss`
3. 边界框损失 `box_loss`
4. 几何点集损失 `point_loss`
5. 普通法线损失 `normal_loss`（对照实验）
6. SOAP 法线监督损失 `soap_normal_loss`（主方法）

总损失可写为：

$$
\mathcal{L}=\mathcal{L}_{percep}+\mathcal{L}_{img}+\mathcal{L}_{box}+\mathcal{L}_{point}+\lambda_n\mathcal{L}_{normal}+\lambda_s\mathcal{L}_{soap-normal}
$$

## 三、数据集与预处理

### 1. 数据集介绍

本文使用 VFHQ 本地子集进行训练，共使用两种版本。

| 数据集 | train | val | test | 用途 |
| --- | --- | --- | --- | --- |
| vfhq_group100_116 | 82346 | 10094 | 0 | 基线与 screen loss 对照 |
| vfhq_group100_101_matted | 8898 | 64 | 32 | matted 与 SOAP 对照 |

### 2. 数据预处理流程

#### 2.1 数据清洗与格式构建

原始数据被构造成 `img_lmdb + optim.pkl + dataset.json` 的训练格式。其中 `img_lmdb` 存储裁剪后的人像图像，`optim.pkl` 存储跟踪后参数，`dataset.json` 存储数据划分信息。

#### 2.2 Matting 预处理

本文新增 `tools/matte_lmdb_dataset.py` 对已有 LMDB 数据进行前景抠图，将背景统一压为黑色，以减轻训练与推理间的背景域偏移问题。

#### 2.3 SOAP 数据生成

本文新增 `tools/export_soap_feature_frames.py`，从每个视频中导出 canonical feature frame，再将这些图像送入 SOAP 离线处理，得到 6 视角图像和法线图，供后续训练读取。

#### 2.4 数据划分与张量构建

训练时，`loader_track.py` 会同时读取 feature image、target image、FLAME 跟踪结果以及 SOAP guidance，并组织成张量 batch 输入模型。

## 四、作品实现

### 1. 总体结构与设计思路

程序整体流程可以概括为：数据处理 -> 模型定义 -> 训练 -> 验证 -> 推理 -> 可视化。

其中，本文在原始项目基础上增加了两条关键能力。

1. 数据链路扩展：支持 matting 数据集和 SOAP 离线多视角先验的接入。
2. 损失函数扩展：支持 normal loss 和 soap normal loss。

### 2. 模型结构设计

虽然本项目不是单纯的 CNN、LSTM 或 Transformer 任务，但从网络模块上仍可分为以下几层。

1. 输入层：输入为 feature image 与 target image。
2. 特征编码层：利用 DINOBase 提取图像语义特征。
3. 几何生成层：通过 lifting planes 和高斯参数生成三维表示。
4. 渲染层：通过 Gaussian renderer 生成基础渲染结果。
5. 上采样层：利用 StyleUNet 输出高质量结果图像。
6. 几何监督层：计算 point loss、screen normal loss 或 SOAP normal loss。

### 3. 核心算法与实现步骤

#### 3.1 库导入与环境配置

项目环境来自 `environment.yml`，核心版本为 PyTorch 2.4.1、PyTorch3D 0.7.8、CUDA 12.1。

#### 3.2 数据集加载与预处理函数

在 `core/data/loader_track.py` 中新增 SOAP guidance 读取逻辑，并修复视频 ID 解析方式，确保同视频采样和 SOAP 路径匹配正确。

#### 3.3 模型类定义

在 `core/models/GAGAvatar/models.py` 中新增 normal loss 与 SOAP guidance 分支，扩展了模型初始化、forward 输出与 `calc_metrics()` 逻辑。

#### 3.4 训练函数与验证逻辑

在 `train.py` 中，训练步数设置为 20000，每 2000 步验证一次，并在验证时输出 PSNR、SSIM 以及拼图图像。同时新增 TensorBoard 标量与图像记录功能。

#### 3.5 评估指标计算

本项目以 PSNR 与 SSIM 为主要量化指标。PSNR 用于反映像素重建误差，SSIM 更强调结构相似性。课程作业展示中以 SSIM 作为 best checkpoint 选择标准。

#### 3.6 结果可视化与保存

训练过程会保存 `examples/*.jpg` 验证拼图，推理结果保存到 `render_results/GAGAvatar/`。拼图的三列分别为目标图像、基础渲染结果和超分结果。

#### 3.7 模型对比实验设计

本文的对比实验分为两组。

1. 同一数据集 group100_116 上：基线 vs screen normal loss。
2. 同一数据集 group100_101_matted 上：matted 基线 vs SOAP 从头训练 vs SOAP 微调。

## 六、结果分析与讨论

### 1. 模型效果分析

模型之所以能够完成任务，主要依赖以下几点。

1. FLAME 提供了稳定的人脸几何先验。
2. DINO 特征与高斯表示能够较好地保留外观信息。
3. matting 处理减少了背景噪声，使训练目标更加集中在人脸主体区域。

### 2. 模型对比分析

screen normal loss 的优点是实现直接、与现有 FLAME 监督兼容；缺点是提升有限。SOAP normal guidance 的优点是提供了额外多视角几何信息；缺点是引入伪标签噪声和优化不稳定。

当前结果表明：如果只看最终量化指标，matted 基线仍然最好；如果看探索价值，SOAP 微调路线更值得继续。

### 3. 误差来源与问题分析

1. SOAP 法线并非真实标注，存在先验误差。
2. 多视角法线与目标相机空间的对齐并不完全精确。
3. 训练步数较少，20000 iter 对复杂几何约束可能不足。
4. 两个实验子集规模差异较大，影响横向结论的严谨性。

## 七、总结与改进展望

### 1. 作品总结

本文完成了 GAGAvatar 的本地复现、训练与推理，并在此基础上加入了 SOAP 多视角法线先验监督与 matting 预处理。项目不仅实现了功能闭环，还进行了多组对比实验，具备较完整的研究与工程实践价值。

### 2. 改进方向

1. 将 SOAP 约束改为分阶段启用，而不是从头训练直接加入。
2. 在同一数据子集上补齐更严格的公平对照实验。
3. 调整法线损失权重与视角组合。
4. 尝试加入 SOAP RGB guidance 与边缘约束。
5. 在更多身份和驱动序列上做泛化测试。
